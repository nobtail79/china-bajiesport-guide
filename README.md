# china-bajiesport-guide

本项目是一个用于归档和发布多个独立 HTML 页面的仓库。页面内容涵盖各类信息展示、工具页面、参考文档等，所有页面均为静态 HTML 文件，可直接通过浏览器访问。

## 项目简介

本仓库旨在以简单、直接的方式存放和共享多个独立的 HTML 页面。每个页面均为自包含的静态文件，不依赖后端服务或数据库。仓库本身不针对任何特定域名或网站，仅作为页面文件的集中存储与版本管理使用。

## 目录结构

```
.
├── pages/       # 存放所有 HTML 页面文件
├── assets/      # 页面引用的静态资源（如图片、样式、脚本等）
├── README.md    # 本说明文件
└── ...
```

- `pages/` 目录下每个子文件夹或文件对应一个独立页面，命名清晰，便于识别。
- `assets/` 目录用于存放页面所需的公共资源，具体引用方式请见各页面内部说明。

## 页面归档说明

所有页面均以 HTML 格式归档，遵循以下原则：

- 每个页面内容独立，不依赖其他页面。
- 页面中不包含任何外部追踪、广告或营销代码。
- 页面样式与功能以简洁实用为主，优先保证在主流浏览器中的正常显示与交互。

如需新增页面，请将 HTML 文件放入 `pages/` 目录，并确保所有资源路径正确。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 来修正页面中的错误或提供改进建议。
- 新增页面请保持文件结构与现有风格一致。
- 不建议在页面中加入动态内容或外部 API 调用，以免影响离线访问或长期可用性。
- 仓库维护者会定期检查页面可用性，但不保证所有页面永久有效。

## 使用方式

您可以直接通过 GitHub Pages 或其他静态托管服务部署本仓库，从而在线访问所有页面。具体部署方式请参考相关平台的文档。
